Scroll Animation Behaviour

How the scroll-driven activation works
Scroll 0%
Section enters viewport → Header fades in. All 4 steps are muted/greyscale.
Scroll 15%
Step 1 activates: Circle fills sage→green gradient. Icon animates from greyscale→colour. Card border turns green, shadow appears. Connecting line starts drawing downward.
Scroll 35%
Step 2 activates: Line reaches step 2 circle. Circle fills teal gradient. "24 Hours" text gets a subtle pulse highlight. Card slides left slightly (x: 10→0).
Scroll 55%
Step 3 activates: Line continues. Circle fills terracotta. Handshake icon does a gentle "shake" animation. Card text transitions from muted→full colour.
Scroll 75%
Step 4 activates: Line completes. Circle fills berry/green. Heart icon pulses once. "Care Begins" text shifts to serif (Cormorant) as a final flourish. All steps now fully active.
Scroll 100%
Section fully scrolled through. All steps remain active. Inline CTA appears: "Ready to start? → Request Care"

Design Decisions

Vertical vs Horizontal

Switch from horizontal cards to vertical timeline. Reads naturally (top→bottom = start→finish). Works beautifully on mobile without layout changes. Scroll activation feels more natural vertically.

Progressive Disclosure

Steps start muted and activate as user scrolls. Creates anticipation and rewards engagement. User literally "walks through" the journey — powerful metaphor for "we walk with you."

Colour Progression

Each step uses a different brand colour (sage → teal → terracotta → berry). Subtly communicates progression and gives each step its own identity within the palette.

Mobile Behaviour

On mobile, uses simpler intersection-observer triggers instead of scroll-linked. Each step fades in when it enters viewport. Same visual result, lighter performance. Line still draws, but via CSS animation on visibility.
