5. Performance & Lighthouse Optimisation

Target: 95-100 across all metrics
Performance (Target: 95+)
  • Images: Convert all to AVIF with WebP fallback. Responsive srcset (640, 768, 1024, 1280). Add explicit width/height to prevent CLS.
  • Fonts: Subset Cormorant + DM Sans to Latin-only glyphs. Use font-display: swap (keep). Preload critical font files.
  • JS: Dynamic import Framer Motion (only load on client). Code-split per route. Tree-shake Lucide icons (import individually, not from barrel).
  • CSS: Tailwind purge is already active. Inline critical above-fold CSS. Defer below-fold styles.
  • LCP: Preload hero image. Use priority on Next/Image. Optimise hero to <100KB.
Accessibility (Target: 100)
  • Focus management: Trap focus in FormModal. Auto-focus first field on open. Return focus on close.
  • Skip link: Fix current implementation (use sr-only + focus:not-sr-only pattern).
  • ARIA: Add aria-describedby to modals. Keyboard escape for dropdowns. Live regions for form status.
  • Contrast: Audit all text/bg combinations. Ensure 4.5:1 minimum on body text, 3:1 on large text.
  • Motion: Respect prefers-reduced-motion everywhere (already partially done, needs audit).
SEO (Target: 100)
  • Already strong: JSON-LD schemas, metadata, breadcrumbs, OG tags.
  • Add: Image alt text audit (descriptive, keyword-rich). Structured FAQ with FAQ schema on service pages.
  • Sitemap: Ensure all 26 routes in sitemap.xml. Add lastmod dates.
Best Practices (Target: 100)
  • HTTPS: Already via Netlify ✓
  • Security headers: Add CSP, X-Content-Type-Options, Referrer-Policy via netlify.toml.
  • Console errors: Audit and fix any console warnings/errors.
  • Passive listeners: Ensure all scroll/touch handlers use passive: true.
